We use our computers (and other devices, which are still just mini-computers), for so many everyday tasks, and it's truly remarkable what goes on under the hood to make everything from writing a paper to paying bills online possible. So many factors come together to allow us to use our computers the way we want to, but a computer's memory is arguably the most crucial to its ability to perform and carry out tasks. Here's a high-level, basic rundown of some of the key players in a computer's memory, and how they work.

What happens when you turn on your computer and use an application? Here's a quick look:

  1. After you turn the computer on, the computer loads everything it needs to do a basic boot-up of its system by running code from its ROM (read-only memory). We'll look at ROM a bit more later on.
  2. The computer loads its operating system from its hard drive into its RAM (random access memory)
  3. You click on your application to open, and the application loads in your computer's RAM. All the work you do in that application will be stored in your computer's RAM.
  4. You save your work, and the data in your file is written to a pre-determined storage device, and deleted from your RAM.

This is a very simplified, high-level look at some of the major players in a computer's memory, specifically RAM and ROM. However, RAM and ROM are larger vehicles that deal with the actual data in your computer's memory. This data, and how your computer stores data in memory, is examined below.

A computer's memory is what holds all of the programs and data that are used by your computer's processor. A processor is what carries out the millions of lines of code that execute programs, the operating system on your computer, etc. When you want to work on something, let's say a sleek Ruby program you're writing, your processor has to pull the file from your hard drive (or flash drive), and store it in memory.

All things, even complex programs and applications, exist in a computer's memory as numbers-- specifically 1's and 0's which make up the binary system (by the way, the subtitle to this post "01101000 01101001" is just "hi" in binary!) A computer's memory has millions and millions of electronic switches called transisters that are flipped on and off when a program is run (really, it's an electrical current that flips them on and off). When you think about the complexity of some of the applications we use on our computers, and the thousands of lines of code that comprise them, the fact that it all comes down to 1's and 0's is truly remarkable.

It's easy to get elbows-deep in binary, specifically the math behind it, but let's get back to how it relates to a computer's memory. We know that data exists in a computer's memory as binary numbers. But how are these numbers organized? Much like we might organize students' names in a class, or shopping list items in an array in a program, a computer's memory is organized in a similar fashion. If you think of a computer's memory as a giant array, each binary number is like an element in the array, and is called a memory location. In an array, each element has an index, indicating it's position in the array. A memory location in a computer's memory has an address. The smallest unit of memory in a computer is a bit, which is represented by either a 1 or a 0. Most computer memories actually work with bytes which are eight bits organized together.

1 byte = 8 bits.

Each byte has an address in a computer's memory. When we write javascript programs, for example, and create a new variable in our program, that variable is stored in a computer's memory as a collection of bytes, which a specific address. When these variables are modified in our programs, let's say with a function, the computer goes to the address associated with that variable, retrieves the value, modifies it according to the function, and stores the changed value in its original address.

When a value is changed, the computer works in a register to change it. You can think of a register as a temporary work station where a computer performs some operations.

We talked about bits being organized into bytes, but bytes can also be grouped into words, which allow computers to store more information in a single place. This allows more information to be retrieved more quickly, as well, since the computer doesn't need to look into as many separate bytes to gather what it's looking for.

Back to RAM and ROM. The vast majority of work you do on a computer is pulled from, and written to, your computer's RAM. The ROM is mostly used to store and execute code that allows the computer to do a basic boot-up. After that, the operating system, applications on your computers, code libraries, etc., are all pulled from your hard drive onto your RAM, where they wait to be executed. It's important to note that RAM is volatile, which means that a computer remembers things in RAM only when it has power. When the power goes off, a computer can't remember what's stored in RAM. ROM is non-volatile-- it contains code pre-programmed and stores the BIOS (basic input/output system) that operates your screen and keyboard.

There are so many different facets that make up the whole of a computer's memory, including caches, logic gates, DRAM, SRAM, etc., and it's easy to find yourself down one of these rabbit holes. The low-level inner workings of a computer's memory isn't always relevant to a high-level programmer's work, but becoming well-versed in a couple of these areas is never a bad idea!

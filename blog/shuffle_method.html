If you ever want to try to understand, or at least really appreciate what goes in to Ruby's beautiful built-in methods, try writing your own! Today I attempted to write a custom shuffle method in Ruby. While I certainly cannot guarantee this is how .shuffle() actually works, it gets the job done.

The first thing that comes to mind when I think of shuffling a collection of information, is "random." We're taking a set of data, whether it's numbers, names, etc., and rearraging the items in the set in RANDOM order. Utilizing Ruby's rand() method will be really important to us in this sense. To start, I create a method called "shuffle," which takes an array. Since I have a feeling that random numbers are going to come into play for the method, I initialize an empty array called random_nums, as seen below.

Think about shuffling an array as taking all of its elements and putting them in random positions in a new array. Therefore, the random numbers that we're about to generate could be considered the NEW INDICES that the original elements of our array will get in a NEW ARRAY.

I create an empty array to store my randomly-generated numbers in. We need to have the same amount of random numbers (random indices) that are in our original array, and we'll need to make sure that we don't generate the same number twice, since each number in the original array (and new, shuffled array) will need to have a unique index.
